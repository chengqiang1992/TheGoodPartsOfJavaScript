JSON

JavaScript对象表示发（JavaScript Object Notation，简称JSON）是一种轻量级的数据交换格式。它基于JavaScript的对象字面量表示法，那是JavaScript最精华的部分之一。尽管只是JavaScript的一个子集，但它与语言无关。所有以现代编程语言编写的程序。都可以用它来彼此交换数据。他是一种文本格式，所以可以被人和机器阅读。它易于实现且易于使用。

安全地使用JSON

JSON特别应用在Web应用中，因为JSON就是JavaScript。使用eval函数可以把一段JSON文本转换成一个有用的数据结构

然后，eval函数有着骇人的安全问题。使用eval去解析JSON文本安全吗？目前，在web浏览器中从服务器端获取数据的最佳技术就是XMLHttpRequest。XMLHttpRequest只能从生成HTML的同源浏览器服务端获取数据。使用eval解析来自那个服务器的文本与解析原来的HTML一样不安全。那是假定该服务器存在恶意的前提下，但如果它只是存在漏洞呢？

有漏洞的服务器或许并不能正确地对JSON进行编码。如果他通过拼凑一些字符串而不是使用一个合适的JSON解码器来创建JSON文本，那么它可能在无意间发送了危险的数据。

在外部数据与innerHTML进行交互时还存在另一种危险。一种常见的Ajax模式是把服务器端发送过来的一个HTML文本片段赋值给某个HTML元素的innerHTML属性。这是一个非常糟糕的习惯。如果这个HTML包含一个标签或其他等价物，那么一个恶意脚本将被执行。这可能也是因为服务器端存在漏洞。

具体有什么危险呢？如果一个恶意脚本在你的页面上运行，它就有权访问这个页面的所有状态和执行该页面能做的所有操作。它能与你的服务器进行交互，而你的服务器将不能区分正当的请求和恶意请求。恶意脚本还能访问全局对象，这使得他有权访问应用中隐藏于比保重的变量之外的所有数据。它可以访问document对象，这会使得他有权访问用户所能看到的一切。它还给这个恶意脚本提供了与用户进行会话的能力。浏览器的地址栏和所有反钓鱼程序会告诉用户这个会话是可靠的。

有漏洞的服务器或许并不能正确地对JSON进行编码。如果他通过拼凑一些字符串而不是使用一个合适的JSON解码器来创建JSON文本，那么它可能在无意间发送了危险的数据。
